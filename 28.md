Hello, Principia 2026.06.06 has been released. It has been over a year since the last release and development has been slow, but enough changes have accumulated since then to warrant a new release.

As usual, you can find downloads on the **[Download](/download)** page, and the full list of changes in [the Changelog](/wiki/Changelog). F-Droid and various Linux distribution packages should receive the update within due time.

## New user data storage!
This one is a big one and is important to mention first of all. The layout for the user data directory has changed, and on some platforms the location has changed as well.

- Linux users can rejoice as the `~/.principia` directory is no more. We now use standard XDG directories, which unless you have defined `XDG_DATA_HOME` to something else will be `~/.local/share/principia`.
- Windows users will find their user data in `%APPDATA%\Principia` (i.e. the `AppData\Roaming` directory) rather than the previous directory right in the user profile directory.

When running the new version with old user data, the game will automatically migrate the data to the new location and directory structure. I've done enough testing to hope things will go smoothly for most, but don't have enough devices to test on to be fully sure. However, the migration process is designed to never cause any data loss, so if something goes wrong you should be able to move things over manually instead.

Once the migration is complete, anything that it didn't move to the new location will be left in a `_old` suffixed directory in the old location, which you can clean up manually if so desired.

See the [User Data Directory Migration](https://principia-web.se/wiki/User_Data_Directory_Migration) wiki page for more details, if migration didn't go as planned, or if you want to clean up old data.

## Gearbox is here! (...sorta)
The [Gearbox](https://principia-web.se/wiki/Gearbox) is an elusive object from early in the game's development, which remained unused in the game's code but for the longest time would crash the game due to the code having bitrotted over the years. However, recently I managed to fix up what was wrong with it, enough that it can be placed into a level without crashing!

## The suction cup model...
In other news, the model of the Suction Cup has been flipped in order to fix the visual representation of the sockets. Previously IN0 looked like an output and OUT0 looked like an input, but this has now been fixed.

	<img src="/news/images/28/suction_cup.webp" style="width:421px" alt="The configuration dialog for the gearbox, showing a very abstract representation of the gearbox axles being drawn with just simple rectangles.">

If you have modelling knowledge and would like to contribute to the game, you may be interested in the newly created [Modelling for the game](https://principia-web.se/wiki/Modelling_for_the_game) page on the Wiki which contains notes on how to make (or edit) models for the game.

## General quality of life improvements
Here are some more improvements that are worth mentioning:

If you try to log in with incorrect credentials, the top-left username text will not update. Previously it did for whatever reason and this would confuse people thinking they were logged in when they were, in fact, not.

<figure>
	<img src="/news/images/28/topleft_username.webp" style="width:700px" alt="The top-left part of the main menu showing the Principia title and my username 'ROllerozxa' in the corner.">
</figure>

On desktop in the LuaScript dialog, there is now a button to take you to the folder where the Lua script file exists if you check "Use external editor", saving you some filesystem navigation. Do note that the file still will not appear until you press save. I tried to also fix this, but turns out there was a reason they made it like that...

<figure>
	<img src="/news/images/28/luascript_external_folder.webp" style="width:800px" alt="Bottom half of the LuaScript GTK dialog. There is a new Open script folder button spanning the entire dialog.">
</figure>

Android has seen some more improvements:

- The navigation bar will now no longer obstruct part of the game area! I only have a phone on Android 12 for testing, so I did not catch this when working on 2025.04.05.
- Fix crash when pressing F11 (fullscreen toggle) on Android. Apparently some phones simulate this key press when touching the fingerprint sensor.
- Clicking on external links in the embedded WebView will now take you to your actual browser instead of trying to open the external website inside the WebView, which usually does not work well.

## Web version
Did you know that Principia now runs in the browser? Since around December of last year, there has been a new "Play in browser <span class="beta">(BETA)</span>" link on level pages which allows you to play any level on the community site right in your browser without needing to download the game.

<figure>
	<img src="/news/images/22/principia_wasm_3.webp" style="width:960px" alt="Screenshot of Principia running with the browser address bar visible at the top.">
	<figcaption><a href="/archive/level/1294">"Art adventure" running in the web version of Principia</a></figcaption>
</figure>

This has been made possible by the game being ported to WebAssembly, something that has been in the works for a long time now. While it is still <span class="beta">BETA</span> and performance may be subpar, it is a great way to try out the game without installing anything, as well as playing the game on platforms where there is no native version available.

Try it!
